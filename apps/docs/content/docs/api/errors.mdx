## 错误响应格式

所有 API 错误返回统一的 JSON 格式：

```json
{
  "error": {
    "message": "详细错误描述",
    "type": "error_type",
    "code": "error_code"
  }
}
```

## 错误码列表

### 401 Unauthorized

**原因**：认证失败

- API Key 无效或已被删除
- 未提供 API Key
- API Key 格式错误

**处理**：检查 API Key 是否正确，确认使用 `Authorization: Bearer xk-xxx` 或 `x-api-key: xk-xxx` 格式。

### 402 Insufficient Credit

**原因**：xtokens 余额不足

- 账户余额为零或不足以支付本次调用

**处理**：通过供应模型赚取 xtokens，或在控制台查看余额详情。

### 404 No Offering Available

**原因**：没有可用的模型供应

- 请求的模型在网络中没有在线的供应节点
- 你没有连接该模型
- 模型名称拼写错误

**处理**：检查模型名称是否正确，确认已在模型网络中连接该模型，查看该模型是否有在线供应节点。

### 429 Rate Limit Exceeded

**原因**：请求频率超过限制

- 默认限制为 60 次请求/分钟

**处理**：降低请求频率，实现请求重试和退避策略。参考响应头中的 `X-RateLimit-Reset` 获取限制重置时间。

### 502 Provider Execution Failed

**原因**：上游供应商执行失败

- 供应商 API 返回错误
- 供应商服务不可用
- 请求超时

**处理**：稍后重试。如果持续失败，可能是供应节点的问题，平台会自动切换到其他可用节点。

## 错误处理建议

```python
from openai import OpenAI, APIError

client = OpenAI(
    api_key="xk-your-api-key",
    base_url="https://api.xllmapi.com/v1"
)

try:
    response = client.chat.completions.create(
        model="deepseek-chat",
        messages=[{"role": "user", "content": "你好"}]
    )
except APIError as e:
    if e.status_code == 402:
        print("余额不足，请充值或供应模型赚取 xtokens")
    elif e.status_code == 429:
        print("请求过于频繁，请稍后重试")
    elif e.status_code == 502:
        print("供应商暂时不可用，请稍后重试")
    else:
        print(f"API 错误: {e}")
```
