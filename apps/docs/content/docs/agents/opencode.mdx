description: 在 OpenCode 中配置 xllmapi 作为 LLM 供应商
---

## 配置文件

在项目根目录创建 `opencode.json`：

```json
{
  "provider": {
    "xllmapi-openai": {
      "name": "xllmapi (OpenAI)",
      "type": "openai",
      "apiKey": "xk-your-api-key",
      "baseURL": "https://api.xllmapi.com/v1",
      "models": {
        "deepseek-chat": {
          "name": "DeepSeek Chat",
          "maxTokens": 8192,
          "contextLength": 64000
        }
      }
    },
    "xllmapi-anthropic": {
      "name": "xllmapi (Anthropic)",
      "type": "anthropic",
      "apiKey": "xk-your-api-key",
      "baseURL": "https://api.xllmapi.com/anthropic/v1",
      "models": {
        "MiniMax-Text-01": {
          "name": "MiniMax Text 01",
          "maxTokens": 16000,
          "contextLength": 1000000,
          "thinking": {
            "type": "enabled",
            "budgetTokens": 10000
          }
        }
      }
    }
  }
}
```

## 说明

- **OpenAI 格式模型**（如 `deepseek-chat`）：使用 `type: "openai"`，Base URL 为 `/v1`
- **Anthropic 格式模型**（如 `MiniMax-Text-01`）：使用 `type: "anthropic"`，Base URL 为 `/anthropic/v1`
- `maxTokens`：模型单次输出的最大 token 数
- `contextLength`：模型的上下文窗口大小
- `thinking`：启用思考过程输出（仅 Anthropic 格式支持）

## 模型限制设置

你可以为每个模型设置 token 限制，避免意外的高额消耗：

```json
{
  "models": {
    "deepseek-chat": {
      "name": "DeepSeek Chat",
      "maxTokens": 4096,
      "contextLength": 64000
    }
  }
}
```

调整 `maxTokens` 即可控制单次输出上限。
