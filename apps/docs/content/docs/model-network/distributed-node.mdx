## 什么是分布式节点

分布式节点是由用户自行部署的供应方式。节点通过 WebSocket 连接到 xllmapi 平台，你的 API Key 始终保留在本地机器上，平台无法看到你的密钥。

适合注重数据隐私、有自己服务器的用户。

## 安装 node-cli

```bash
npm install -g @xllmapi/node-cli
```

## 配置与启动

### 获取节点 Token

1. 在控制台的「供应管理」中点击「创建分布式节点」
2. 系统生成一个节点 Token
3. 复制保存该 Token

### 配置文件

创建配置文件 `xllmapi-node.json`：

```json
{
  "token": "your-node-token",
  "providers": [
    {
      "name": "deepseek",
      "apiKey": "sk-your-deepseek-key",
      "baseUrl": "https://api.deepseek.com",
      "models": ["deepseek-chat", "deepseek-reasoner"]
    }
  ]
}
```

### 启动节点

```bash
xllmapi-node start --config xllmapi-node.json
```

节点启动后会：

1. 通过 WebSocket 连接到平台
2. 自动发现配置的模型
3. 将模型发布到网络中

## 工作流程

```
用户请求 → xllmapi 平台 → WebSocket → 你的节点 → LLM 供应商
                                        (本地解密 API Key)
```

平台只负责路由请求，实际的 API Key 使用和请求转发都在你的节点本地完成。

## 数据隐私

- 你的 API Key **永远不会离开本地机器**
- 平台仅传递用户的请求内容和模型响应
- 你可以随时审计节点的网络流量

<Callout type="warn">
分布式节点的可用性取决于你的本地运行状态。如果节点离线，通过该节点供应的模型将不可用。
</Callout>
