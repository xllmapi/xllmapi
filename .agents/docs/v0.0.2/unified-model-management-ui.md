# 统一模型管理 UI 设计

> 版本: v0.0.2 | 日期: 2026-03-23

## 问题

供应商管理分为两个独立页面：NetworkPage（上传 Key）和 NodesPage（分布式节点），割裂感强。消费者还有单独的偏好设置页。

## 方案

统一为一个「模型管理」页面 (`/app/models`)，两个 Tab：

```
模型管理:
  ├─ Tab 1: 使用的节点（消费者视角）
  │   → 从市场收藏的模型节点 = 我的路由池
  │   → 发请求时从这个列表中路由
  │
  └─ Tab 2: 提供的节点（供应商视角）
      → L1(上传Key) + L3(本地节点) 混合展示
      → [+ 接入新模型] → 选择方式 → 走对应流程
      → 节点 Token 管理（L3 用户可见）
```

## 关键决策

1. **收藏 = 使用列表 = 路由池** — 三合一，`offering_favorites` 表复用
2. **砍掉** `user_connection_pool` 和 `user_node_preferences` 表
3. **砍掉** `NodePreferencesPage` — 不需要单独偏好设置
4. **L1/L3 标签区分** — `☁️ 平台托管` / `🖥️ 本地节点`，同一列表展示
5. **侧边栏一个入口** — 「模型管理」替代之前的三个入口
