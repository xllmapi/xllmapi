# 统一模型管理 UI 设计

> 版本: v0.0.2 | 日期: 2026-03-23

## 问题

供应商管理分为两个独立页面：NetworkPage（上传 Key）和 NodesPage（分布式节点），割裂感强。消费者还有单独的偏好设置页。

## 方案

统一为一个「模型管理」页面 (`/app/models`)，两个 Tab。

---

### Tab 1: 使用的节点（消费者视角）

```
我添加的节点 (3)                          [去市场添加 →]
  活跃 (2)
  ┌ mn_x9zQ  deepseek-chat  @user_abc  ☁️托管   🟢  280/450 xt
  └ mn_jT8w  deepseek-chat  @gpu_guy   🖥️分布式  🟢  100/200 xt

  不活跃 (1)
  ┌ mn_pK7s  qwen2:7b  @home_miner  🖥️分布式  ⚫离线  2小时前

────────────────────────────────
平台模型 (默认可用)
  ┌ deepseek-chat      3个节点  🟢  均价 ~320/480 xt
  ├ gpt-4o             2个节点  🟢  均价 ~950/1900
  └ claude-sonnet      1个节点  🟢  均价 ~1500/3000
  ℹ️ 按模型合并，近 7/30 天均价，所有用户默认可用
```

**设计要点**：
- **收藏 = 使用列表 = 路由池**（三合一）
- 托管和分布式混合展示，☁️/🖥️ 标签区分，分活跃/不活跃
- 平台模型按模型名合并（背后多个供应商），显示均价和可用节点数，不可移除
- 用户托管和平台托管本质一样（都是 ☁️ 平台调用），区别只是谁买的 Key
- **路由优先级**：用户收藏节点 → fallback 平台模型

---

### Tab 2: 提供的节点（供应商视角）

```
活跃 (2)                                    [+ 接入新模型]
  ┌ mn_aB3k  deepseek-chat  ☁️平台托管  🟢运行中  请求:1234  收入:50K
  └ mn_jT8w  llama3:8b      🖥️本地节点  🟢在线    请求:456   收入:12K

不活跃 (3)
  ┌ mn_qR5n  gpt-4o     ☁️平台托管  ⏸️已暂停
  ├ mn_mN3v  mistral:7b 🖥️本地节点  ⚫离线  上次在线: 5小时前
  └ (未命名)  qwen2:72b  🖥️本地节点  📎未接入网络  [发布到网络]
```

**状态定义**：
- 🟢 运行中 — offering 上线，节点在线，正在服务 → 活跃
- ⏸️ 已暂停 — 供应商手动暂停 → 不活跃
- ⚫ 离线 — 节点断线或 Key 失效 → 不活跃
- 📎 未接入网络 — 节点已连接平台但未发布到网络 → 不活跃
- 🟡 待审核 — 已提交发布，等待管理员审核 → 不活跃
- ⬜ 已停止 — 供应商主动下架，保留历史 → 不活跃

---

## 关键决策

1. **收藏 = 使用列表 = 路由池** — `offering_favorites` 表复用
2. **砍掉** `user_connection_pool` 和 `user_node_preferences` 表
3. **砍掉** `NodePreferencesPage` — 收藏即路由池，不需要单独偏好
4. **☁️/🖥️ 标签区分执行方式** — 不按类型分组，混合展示
5. **平台模型合并** — 按模型名合并，显示均价和节点数，不暴露 offering ID
6. **用户 offerings 独立** — 每个有 mn_ ID，独立评价/信誉
7. **侧边栏一个入口** — 「模型管理」替代之前三个入口
8. **自动降级** — 节点离线自动移到不活跃，重新上线自动回活跃
