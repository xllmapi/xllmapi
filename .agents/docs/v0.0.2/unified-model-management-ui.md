# 统一模型网络架构 + UI 设计

> 版本: v0.0.2 | 日期: 2026-03-23

## 核心决策：所有模型节点一视同仁

平台自己也是一个供应者账号（@xllmapi-official），通过同样的流程发布模型。
没有"平台模型"这个特殊概念。每个模型节点都有独立 mn_ ID、独立评价。

### 冷启动：新用户注册时自动关注 @xllmapi-official

---

## 消费者视角

### 列表卡片（简约，点击进入详情）

```
┌──────────────────────────────────────────────────────────┐
│ 🟢 DeepSeek 高速节点          ✅已验证   ⭐4.9          │
│ deepseek-chat · @xllmapi-official       300/500 xt/1K   │
└──────────────────────────────────────────────────────────┘
```

### 详情页（/mnetwork/:nodeId）

- 节点名 + 模型名 + 供应者 + 在线状态
- 真实性验证状态 + 价格 + 认证/押金
- **唯一操作按钮：「加入使用列表」**
- 运行数据：**可视化图表**（在线率趋势、成功率趋势、延迟分布、请求量趋势）
- 评论区

### 消费者不看到接入方式

☁️ 托管 / 🖥️ 分布式只在供应者控制台显示。
消费者看到的是**模型真实性认证程度**：
- ✅ 已验证 — 平台检测确认
- 🟡 待验证 — 新上线
- ⚠️ 存疑 — 与声称不符
- ℹ️ 供应者声称 — 未验证

---

## 供应者视角

### 模型管理（/app/models）

两个 Tab：
- **使用的节点**：我收藏的（= 路由池），活跃/不活跃分区
- **提供的节点**：我发布的（☁️/🖥️ 标签），活跃/不活跃分区，含节点 Token 管理

---

## 模型市场（/mnetwork）

复用现有页面，增加：搜索 + 筛选（验证程度、在线、价格） + 排序
所有节点平铺，不分组。用户自己搜索和筛选。

---

## 关键简化

1. **收藏 = 使用列表 = 路由池**（三合一）
2. **砍掉** user_connection_pool、user_node_preferences 表
3. **砍掉** NodePreferencesPage
4. **路由只查收藏列表**，无 fallback 逻辑
5. **操作按钮只保留一个**（加入使用列表）
6. **运行数据用可视化图表**，不用数字格子
