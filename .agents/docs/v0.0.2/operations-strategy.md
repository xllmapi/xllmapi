# xllmapi 平台运营策略 — 零成本起步方案

> 版本: v0.0.2 | 日期: 2026-03-23

## Context

平台初期没有 GPU 预算，无法自营开源模型。核心策略：以 L1（托管 Key）和 L3/L4（分布式节点）两条零成本路径起步，平台本身不承担 API 或算力费用。后续随着收入增长，逐步引入 L2 自营和厂商合作。

---

## 核心原则：透明 + 用户自主选择

**平台不替用户做安全判断，提供充分信息让用户自己选择。**

1. **透明告知** — 每种模式的风险明确说明，不做虚假安全承诺
   - L1 托管：明确告知"平台在请求处理时接触明文 Key"
   - L3 分布式：明确告知"节点供应商可看到消息内容"
   - E2E 加密：明确告知"平台不可见，节点可见"
2. **信息公开** — 节点的认证状态、押金、信誉分、评价全部公开，消费者自行评估
3. **不强制分级** — 平台不替用户划定"安全/不安全"，展示数据让用户判断
4. **用户做主** — 用哪个节点、是否 opt-in 分布式、接受什么风险，都是用户自己的决定

---

## 起步策略：两条路并行

```
L1 托管 Key（存 Key 模式）          L3/L4 分布式节点
─────────────────────          ─────────────────
用户上传自己的 API Key              用户本地运行节点
平台加密存储 + 代理调用              Key 留在用户本地
平台成本: 零                       平台成本: 零
消费者使用方便                      供应商 Key 最安全
```

两条路不矛盾，服务同一个市场的不同用户：
- 懒得跑节点的供应商 → L1 上传 Key
- 注重 Key 安全的供应商 → L3 跑节点
- 有本地 GPU 的用户 → L4 跑本地模型
- 消费者 → 不关心后端是 L1 还是 L3，只要模型能用

---

## 第一阶段：上线验证（0-2 个月）

**目标**：让第一批用户跑通完整流程

### 平台自购过渡供给（冷启动关键）
- 平台自费购买少量厂商 API 账号（DeepSeek、MiniMax 等价格低的优先）
- 以平台官方身份发布 offerings，标记为「官方」
- 目的：新用户注册后立即有模型可用，不需要等第三方供应商入驻
- 定价：成本价或微利，不以此赚钱，核心是留住用户
- 预算控制：按月控制在可承受范围（如 $50-100/月），设置单用户日限额
- 随着第三方供应商增多，官方 offerings 的流量占比自然下降
- 本质是「平台补贴期」，类似打车平台初期补贴司机和乘客

### L1 托管（第三方供应商）
- 平台支持用户上传 DeepSeek、OpenAI、Anthropic 等 Key
- 加密存储（AES-256-GCM），已实现
- 用户创建 offering 设定价格 → 审核通过 → 消费者可用
- 对外沟通：明确告知加密机制，不夸大安全性

### L3/L4 分布式
- node-cli 已实现，标记为 Beta
- 支持远程 API（DeepSeek 等）+ 本地模型（Ollama）
- 邀请 10-20 个技术用户测试
- 收集反馈迭代

### 市场基础功能
- 模型市场页面（已实现）
- 投票/收藏/评论（已实现）
- 消费者 opt-in 节点偏好（已实现）

### 增长
- 邀请制（已有），初始赠送 100 万 xtoken
- 在开发者社区推广（V2EX、掘金、GitHub）
- 核心卖点：一个 API 访问所有模型 + 分享 Key 赚 xtoken

**指标**：50+ 用户，10+ offerings，5+ 节点

## 第二阶段：供给扩展 + 信任建设（2-6 个月）

**目标**：扩大供给，建立信用背书体系

### L1 信任升级
- 开源加密/解密相关代码，接受社区审查
- 发布安全白皮书（加密机制、审计日志、最小权限）
- 有量之后主动联系 DeepSeek 谈合作（他们 ToS 宽松，门槛低）

### L3/L4 正式化
- node-cli 从 Beta 转正式版
- 上线节点信用背书体系（用户自由选择参考）：
  - 普通节点：无认证，信誉分 + 投票作为唯一参考
  - 认证节点：身份认证（手机/实名）→ 展示"已认证"标签
  - 押金节点：缴纳 xtoken 押金 → 展示押金金额，违规扣除
  - 消费者自行判断：看信誉分、认证状态、押金金额、评论，自己决定信不信
- 上线消息端到端加密（E2E）：
  - 消费者和节点协商临时密钥
  - 平台只转发密文，不看消息内容
  - 明确告知：防君子不防小人（节点解密后技术上能看到）
- 隐私级别路由：
  - 消费者可选 "仅平台托管" 或 "包含分布式节点"

### 社交生态
- 供应商主页活跃
- 评论/投票形成口碑
- 优质供应商浮现

### xtoken 初步闭环
- 消费者：注册送 xtoken → 用完了需要充值（或自己当供应商赚）
- 供应商：赚的 xtoken 可以消费其他模型
- 暂不开放提现（法律复杂），先做 xtoken 内循环

**指标**：500+ 用户，50+ offerings，30+ 节点，E2E 加密上线

---

## 第三阶段：生态成熟（6-12 个月）

**目标**：飞轮运转，开始有收入

### L1 合作升级
- 与 1-2 家厂商（DeepSeek、MiniMax）签正式合作
- L1 从"灰色共享"变为"授权分销"
- 平台拿到批量折扣，差价作为额外收入

### L2 引入（此时有收入支撑）
- 用平台收入（15% 分成积累）租 GPU 跑热门开源模型
- 自营 offerings 作为"平台推荐"，稳定性最高
- 不需要大规模投入，1-2 台 A100 跑 2-3 个模型即可

### L3/L4 规模化
- 200+ 活跃节点
- 信用背书体系成熟：身份认证 + 押金 + 信誉分 + 用户评价 多维度参考
- 消费者根据自身需求自由选择节点（平台不做强制分级，只提供信息）
- 供应商开始有稳定 xtoken 收入

### xtoken 经济
- 评估开放充值/提现
- 或引入兑换：xtoken → 平台增值服务（更高配额、优先路由等）

**指标**：2000+ 用户，200+ 节点，xtoken 内循环活跃，平台有正收入

## 分级策略演进

```
第一阶段（起步）:
  不分级，所有请求混合路由
  消费者可选 "仅平台托管" 或 "包含节点"（二选一）

第二阶段（扩展）:
  节点信用背书上线: 身份认证 + 押金 + 信誉分
  + E2E 消息加密
  + 消费者自由选择（平台提供信息，不做强制分级）

第三阶段（成熟）:
  消费者看到的节点信息:
  ├─ 信誉分: 4.8/5.0 (基于在线率/成功率/用户投票)
  ├─ 认证状态: 已实名 / 未认证
  ├─ 押金: 50,000 xtoken（违规可扣除）
  ├─ 服务记录: 累计 1,234 次请求，成功率 99.2%
  ├─ 用户评价: 点赞 89 / 点踩 2
  └─ 消费者自己决定是否使用这个节点
```

## 零成本运营的关键

| 项目 | 成本 | 谁承担 |
|------|------|--------|
| API 调用费 | 供应商的 Key | 供应商 |
| GPU 算力 | 供应商的硬件 | 供应商 |
| 服务器 | 平台路由/结算 | 平台（最小化：单台 VPS 即可） |
| 带宽 | WS 转发 | 平台（流量不大） |

平台唯一的成本：一台 VPS 跑平台 API + PostgreSQL + Redis。预估月成本 $20-50。

**收入来源**：15% 结算分成。消费者每花 100 xtoken，平台得 15。

---

## 风险与应对

| 风险 | 应对 |
|------|------|
| 冷启动无供给 | 创始团队自己上传 Key 做第一批供应商 |
| xtoken 无人充值 | 初期赠送足够体验；供应商赚的 xtoken 可消费其他模型 |
| 节点不稳定 | 断路器 + fallback 到其他 offering |
| 用户不信任平台存 Key | 明确告知风险 + 引导试用分布式节点 |
| 消费者担心节点隐私 | 明确告知情况 + 默认关闭节点 + 展示节点信用信息由用户自行决定 |

---

## 阶段检查点

- 第一阶段末：完整流程跑通（上传Key→创建offering→消费者调用→结算）
- 第二阶段末：E2E 加密上线 + 节点信用背书可用 + 500 用户
- 第三阶段末：首个厂商合作签约 + L2 自营上线 + 正收入
